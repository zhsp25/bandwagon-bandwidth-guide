## 搬瓦工双向流量是什么意思？

**搬瓦工双向流量**，指的是 VPS 的出站流量（Outbound）和入站流量（Inbound）**两者之和**都会被计入月度配额。

不是只算"别人下载你服务器上的东西"，而是"你服务器和外部互相传输的所有字节"全部计费。

搬瓦工官方客服对此的表述是：

> *"Bandwidth is calculated with both inbound and outbound traffic."*

这一规则无一例外，适用于所有套餐——包括香港 CN2 GIA、日本东京 CN2 GIA、CN2 GIA-E，以及最入门的 KVM 套餐。

## 为什么同样 1GB 的文件，流量却跑掉了 2GB？

这是大家最困惑的地方，用个具体场景解释。

你通过搬瓦工 VPS 代理，观看一段 1GB 的视频。

实际发生的过程是这样的：

1. 视频源服务器把 1GB 数据**发送进**你的 VPS（入站 1GB）
2. 你的 VPS 再把这 1GB 数据**转发到**你的本地设备（出站 1GB）

结果：1GB 内容 = 2GB 流量消耗。

根据banwago.net 的实测数据，下载一个约 1GB 的文件，KiwiVM 面板显示流量增量约为 **2.06GB**——完全印证了双向计费规则。代理用户估算流量需求时，要按"**实际数据量 × 2**"来计算。

## 建站 vs 代理：两种使用场景的流量消耗差距有多大？

这里有一个很重要的区别，很多人混淆了。

**建站场景**

访客点开你的网页，发出一个几十 KB 的请求（入站），你的 VPS 返回几 MB 的页面内容（出站）。入站量相对于出站量微乎其微，体感接近"单向计费"。搬瓦工官方也提到，建站场景下流量主要来自出站，相对省。

**代理/中转场景**

数据进来再出去，一进一出对等消耗。你用了多少流量，就要乘以 2 来估算配额占用。

所以结论很直接：**同一个套餐，建站用户和代理用户的体感流量差距可以是一倍**。

## 5 个实用技巧，帮你在双向计费下省出更多流量

搬瓦工双向计费的规则改不了，但用法上还是有一些省量空间。

**1. 合理设置代理协议压缩**

部分代理协议支持传输层压缩，对于文本类内容（网页、API 接口）可以减少实际传输字节数，间接降低流量消耗。

**2. 建站用户开启 CDN**

把静态资源（图片、JS/CSS 文件）挂到 CDN 上，访客直接从 CDN 节点取资源，不经过你的 VPS，出站流量可以明显下降。

**3. 开启 KiwiVM 流量告警**

在 KiwiVM 面板里设置 80% 和 90% 告警邮件。流量快耗尽前收到提醒，避免突然停机造成损失。

**4. 避免在 VPS 上中转大文件**

把大文件传到 VPS 再分发出去，进出各算一遍，是流量消耗最大的操作。如果只是备份或临时存放，在月底流量重置前再操作更划算。

**5. 升级套餐而不是等重置**

搬瓦工支持套餐升级，升级后流量立即重置。如果当月还有重要任务要跑，与其干等重置日，不如直接升级到更大的套餐。
